# Typography

Our typography system is built on a consistent scale that ensures readability and visual hierarchy across all components. We use system fonts for optimal performance and native feel.

## Font Families

Our typography system includes four font families for different use cases:

### Sans Serif (Primary)
```css
font-family: var(--font-sans);
```
- **Usage**: Primary text, UI elements, body content
- **Stack**: System UI, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, 'Noto Sans', sans-serif
- **Characteristics**: Clean, modern, highly readable

### Display
```css
font-family: var(--font-display);
```
- **Usage**: Headings, large text, hero sections
- **Stack**: System UI, -apple-system, BlinkMacSystemFont, 'Segoe UI', Roboto, 'Helvetica Neue', Arial, sans-serif
- **Characteristics**: Optimized for larger sizes, slightly more condensed

### Serif
```css
font-family: var(--font-serif);
```
- **Usage**: Special content, quotes, decorative text
- **Stack**: ui-serif, Georgia, Cambria, "Times New Roman", Times, serif
- **Characteristics**: Traditional, readable, elegant

### Monospace
```css
font-family: var(--font-mono);
```
- **Usage**: Code, data, technical content
- **Stack**: ui-monospace, SFMono-Regular, Menlo, Monaco, Consolas, "Liberation Mono", "Courier New", monospace
- **Characteristics**: Fixed-width, technical, code-friendly

## Accessibility Guidelines

### Contrast Requirements

All text must meet WCAG AA contrast requirements:

- **Normal text**: Minimum 4.5:1 contrast ratio
- **Large text (18px+)**: Minimum 3:1 contrast ratio
- **UI components**: Minimum 3:1 contrast ratio

### Readability

- **Line height**: Minimum 1.4 for body text, 1.2 for headings
- **Font size**: Minimum 16px for body text on mobile devices
- **Line length**: 45-75 characters for optimal readability
- **Spacing**: Adequate spacing between lines and paragraphs

### Screen Reader Support

- Use semantic HTML elements (h1-h6, p, strong, em, etc.)
- Provide alternative text for images
- Use proper heading hierarchy
- Ensure focus indicators are visible

## Responsive Typography

Typography scales appropriately across different screen sizes:

### Mobile (320px - 768px)
- **H1**: 2rem (32px)
- **H2**: 1.75rem (28px)
- **H3**: 1.5rem (24px)
- **Body**: 1rem (16px)

### Tablet (768px - 1024px)
- **H1**: 2.5rem (40px)
- **H2**: 2rem (32px)
- **H3**: 1.75rem (28px)
- **Body**: 1rem (16px)

### Desktop (1024px+)
- **H1**: 3rem (48px)
- **H2**: 2.5rem (40px)
- **H3**: 2rem (32px)
- **Body**: 1.125rem (18px)

## Best Practices

### Do's
- Use consistent font sizes from the typography scale
- Maintain proper heading hierarchy (h1 → h2 → h3)
- Ensure adequate contrast ratios for all text
- Use semantic HTML elements for better accessibility
- Test typography on different devices and screen sizes

### Don'ts
- Don't use more than 3-4 different font sizes on a single page
- Don't use font sizes smaller than 12px
- Don't use decorative fonts for body text
- Don't rely solely on color to convey meaning
- Don't use all caps for extended text (reduces readability)
