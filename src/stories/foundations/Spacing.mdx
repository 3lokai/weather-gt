# Spacing & Radius

Our spacing system provides consistent spacing and border radius values throughout the application. All spacing values are based on a visual parity scale (2/4/6/8/12/16/24/32...) for optimal visual harmony and consistency. The system includes comprehensive spacing utilities and weather-specific spacing patterns.

## Spacing Scale

## Responsive Spacing

Spacing scales appropriately across different screen sizes:

### Mobile (320px - 768px)
- **Section padding**: 16px - 24px
- **Component padding**: 12px - 16px
- **Element spacing**: 8px - 12px
- **Grid gaps**: 12px - 16px

### Tablet (768px - 1024px)
- **Section padding**: 24px - 32px
- **Component padding**: 16px - 24px
- **Element spacing**: 12px - 16px
- **Grid gaps**: 16px - 24px

### Desktop (1024px+)
- **Section padding**: 32px - 64px
- **Component padding**: 24px - 32px
- **Element spacing**: 16px - 24px
- **Grid gaps**: 24px - 32px

## Accessibility Considerations

### Touch Targets

Ensure adequate spacing for touch targets:

- **Minimum touch target**: 44px × 44px
- **Recommended touch target**: 48px × 48px
- **Touch target spacing**: Minimum 8px between targets

### Focus Indicators

Provide adequate spacing for focus indicators:

```css
/* Focus ring spacing */
.focusable {
  outline-offset: var(--spacing-1); /* 2px */
}

/* Focus ring size */
.focusable:focus {
  outline: 2px solid var(--ring);
  outline-offset: var(--spacing-1); /* 2px */
}
```

## Best Practices

### Do's
- Use consistent spacing values from the scale
- Maintain visual rhythm with consistent spacing patterns
- Use larger spacing for more important content
- Test spacing on different screen sizes
- Use CSS custom properties for all spacing values

### Don'ts
- Don't use arbitrary spacing values
- Don't mix different spacing patterns inconsistently
- Don't use spacing that's too tight for touch targets
- Don't ignore responsive spacing considerations
- Don't use spacing that breaks visual hierarchy
